Why this is a wizard

It should guide, not just collect fields

Start with people, not prompts The user adds who matters first, so later script and keyframe steps can reuse the same cast data.
Show missing dependencies early If no reference image exists, the wizard can tell the user before they hit the keyframe step.
Keep the existing content flow The same cast names, slugs, and role options already used by the app can drive the mock.
⚡

Suggested prototype behavior

Small enough to evaluate quickly

1. Add one or more people.
2. Assign a reference image and role.
3. Generate the cast-aware brief.
4. Reuse the same cast in keyframe generation.
5. Warn if the selected model needs a reference image.
