Practical requirements.

Requirements depend on the workflows you choose. That is the point: ComfyStudio can be lightweight, heavy local, cloud-first, or hybrid depending on your machine and project.

For lighter local work

Use lower-VRAM local workflows and draft passes when you want fast iteration on consumer hardware. Workflow Setup will show what each graph needs before you queue.

For heavy local video

Larger video workflows can require significant VRAM and disk space for model weights. High-end NVIDIA cards are best for WAN and LTX-style local video stacks.

For weaker GPUs

Cloud workflows let users generate premium shots or upscale videos without forcing every project through local hardware. Credits and API keys depend on the provider.

Common questions.

Do I need ComfyUI?

Yes for local generation. ComfyStudio is a desktop creative workstation that talks to a local ComfyUI server, manages bundled workflows, and keeps generated outputs inside your project.

Can I use my existing ComfyUI install?

Yes. Use Bring Your Own ComfyUI, set the local port and ComfyUI folder, then run Workflow Setup to check nodes and models for the built-in workflows.

Does everything run locally?

Local workflows run through your ComfyUI install. Cloud workflows use provider APIs and credits. You can mix local and cloud outputs in the same project timeline.

How is this different from a single-model video app?

Single-model apps are excellent for generating clips with one engine. ComfyStudio is built around the whole video: multiple models, Director Mode, Flow AI, asset management, timeline editing, effects, upscaling, and export.

Where do generated files go?

Generated media is imported into the current ComfyStudio project so you can review, organize, edit, and export without chasing files across output folders.
